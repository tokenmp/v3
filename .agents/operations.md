# 操作约束

## 环境确认

- 操作前必须明确目标环境：本机、开发/预览服务器或生产服务器。
- 必须明确项目版本、服务器别名、用户、工作目录、容器、数据库和端口，不得根据名称猜测。
- 服务器地址、SSH 用户、部署路径和实时运行状态属于本地私有信息，应从可选的 `.agents/local.md` 及其索引文档读取。
- 本地私有文档不存在或信息不足时，应向用户确认或执行只读检查，不得自行推测。

## 基础设施边界

- 开发、预览和生产环境不得混用。
- 操作 Docker 前核对 Compose project、service、working directory、mount 和资源标签。
- TokenMP v3 Docker 资源遵循 `.agents/docker.md`，统一使用 `tokenmp-v3` 标识。
- 公共数据库、缓存、代理、面板和可观测性组件可能被多个项目使用，不得按单一应用资源处理。
- 涉及停止、重启、部署、迁移、删除或数据修改时，先确认环境和影响范围。

## 数据安全

- 未确认目标数据库、数据所有权和备份情况前，不执行破坏性 SQL、删除容器卷或清理数据。
- 不读取、输出或提交与任务无关的密钥、令牌、密码、证书、生产数据或用户数据。
- 使用凭证时只确认其来源和存在性，不在日志、命令输出或文档中暴露具体值。

## 私有信息

以下信息不得写入可提交的 `AGENTS.md`、`.agents/*.md`、README 或其他公开文档：

- 公网或私网 IP、未公开域名和服务器资产信息。
- 私有 SSH 用户、认证方式和用户级配置。
- 生产数据库名称、账号、密码、连接串和内部端口映射。
- 实时容器清单、内部部署路径和基础设施拓扑。
- 到期时间、费用、供应商账号及其他资产信息。

这类信息存放在被 Git 忽略的 `.agents/local.md`、`.agents/*.local.md` 或用户目录私有文档中。公开规则只描述角色、约束和查找方式。
