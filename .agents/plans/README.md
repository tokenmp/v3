# Agent 临时计划

本目录存放 Agent 为具体任务创建的临时计划、调研笔记、迁移清单和执行记录。

## 适合存放

- 开发实施计划
- 重构或迁移计划
- 排障假设与调查步骤
- 分阶段任务清单
- 发布、回滚或数据迁移演练计划
- 尚未确认的方案草案

## 不适合存放

- 长期有效的项目规范；应放在 `.agents/*.md`
- 已确认的架构决策；应放在正式 ADR 或架构文档
- 项目公共文档；应放在 `docs/` 或 `README.md`
- 密钥、令牌、密码、生产数据或其他敏感信息
- 需要进入 Git 历史的交付文档；具体任务计划默认不提交，应迁移到 `docs/` 或 ADR

## 命名

使用小写英文和连字符：

```text
<topic>.md
<topic>-migration.md
<topic>-investigation.md
```

不要使用 `temp.md`、`plan.md`、`new.md` 等无法识别主题的名称。

## 生命周期

1. 创建时写明状态、目标、范围和完成条件。
2. 执行过程中更新实际进度、决策和阻塞项。
3. 计划完成后，将长期有效的结论同步到正式规则或项目文档。
4. 已完成且没有审计价值的计划应删除。
5. 仍有审计价值的内容应精简后迁移到正式文档，并明确标为历史记录。
6. 不得让已完成的临时计划继续充当现行操作指引。

建议计划头部：

```md
# <计划名称>

- 状态：draft | approved | in-progress | blocked | completed | cancelled
- 创建日期：YYYY-MM-DD
- 最后更新：YYYY-MM-DD
- 关联分支：<branch-name 或 none>
- 负责人：Agent / User

## 目标

## 范围

## 非目标

## 前置条件

## 实施步骤

## 验证

## 风险与回滚

## 决策与阻塞项

## 完成后的文档同步
```
