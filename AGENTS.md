- Executor Phase 10 HTTP streaming：`internal/stream`/`internal/streamfacade` 的 transport-neutral boundary/facade 和 `execution.StreamDriver` 已由 `internal/transport/executorv1api.Hybrid`、SSE sink 与 `internal/composition` 接入 runtime。公开 OpenAI Chat 与 Anthropic Messages `stream:true` 已启用：generated plain handler 在 bounded structural stream detection 后分流，`stream:false` 保持 generated strict non-stream 路径，`stream:true` 只完整 normalization 一次后进入 stream facade/Driver 和精确 OpenAI/Anthropic `StreamClient` registry。`AuthMiddleware` 保持在 `CaptureRawBody` 外层；commit 前失败返回协议原生 JSON，commit 后绝不追加 JSON fallback。SSE sink 需要 flusher、canonical payload 限制 256 KiB；OpenAI framing 以唯一 `[DONE]` 结束，Anthropic 使用 native `event:` + `data:`、无 `[DONE]`。Driver 仍为 pre-commit-only retry、一次 Reserve/唯一 Finalize 或 Release；不声称 HTTP atomicity、wire-attempt proof、跨进程 exactly-once 或 public/provider E2E。Responses non-stream+stream 已 runtime 启用；Images legacy completion-only non-stream 已执行。CI 显式 race 覆盖 `./internal/stream/...`、`./internal/streamfacade/...` 与独立 shared validator `./internal/imagecontract/...`；route conformance/process tests 覆盖 stream pre-commit JSON 404。模块规则：`services/executor/internal/{stream,streamfacade,streaming,sdk,imagecontract}/AGENTS.md`。
- Executor Phase 11 Images HTTP：legacy OpenAI Images 已经 execution registry、composition 和 `/v1/images/generations` non-stream transport 执行。`internal/imagecontract` 为 SDK/transport 共用 validator：untrimmed nonempty prompt、1 MiB prompt、512-byte CTL-free user、default wire/renderer `url`，以及 16 MiB wire、10 MiB item、12 MiB aggregate、usage/extensions/revised-prompt 边界；base64 以 streaming decoder 验证。所有 Images 成功/失败均 `Cache-Control: no-store`。不支持 GPT Image 特有参数或 usage quota；Responses non-stream+stream 已 runtime 启用。
- Executor Phase 12 typed quota domain：`services/executor/internal/quota/AGENTS.md`。typed `Repository`、safe bounded reservation metadata/estimate/terminal settlement、`DomainInMemory` 与 `TypedMock` contract/race/fuzz suite；legacy `Port`/`InMemory`/`Mock` 已删除，`Repository`/`DomainInMemory`/`TypedMock` 是唯一实现。Runner 使用 `AccountingConfirmedUsage`（`sdk.Completion.Known=true`+`Valid()` 时携带 `ConfirmedUsage`）或 `AccountingUnpricedSuccess`；StreamDriver 使用 `AccountingConfirmedUsage`（`streaming.UsageKnown` 时携带 `ConfirmedUsage`）或 `AccountingUnpricedSuccess`。Phase 12.3 non-stream usage normalization：`sdk.Completion` 新增 `Usage`/`Known` 字段，OpenAI Chat 与 Anthropic Messages adapter 从 `RawJSON` 提取 bounded token 计数（≤ 1e6，一致），不合格时 `Known=false`；Images 不提取。Runner 经 `runnerFinalizeOutcome` 映射，与 StreamDriver `confirmedUsage` 对称。无 usage charging、数据库或 durable storage。
- Executor Phase 13 models catalog：`services/executor/internal/modelcatalog/AGENTS.md` 与 `services/executor/internal/modelcatalogfacade/AGENTS.md`。`internal/modelcatalog` 定义 transport-neutral `CatalogProvider` 端口、`CatalogRequest`/`CatalogResult`/`CatalogEntry`/`ThinkingConfig` 形状、secret-free `Principal` 与安全 sentinel，以及 adapter Capability→contract string tag 映射（Chat→`text`、Tools→`function_calling`、Vision→`vision`、Thinking→`thinking`、Images→`image`，streaming/messages/responses 省略）与 ThinkingInput→ThinkingConfig 映射；`internal/modelcatalogfacade` 实现 `CatalogProvider`：per-request pin 当前 compiled snapshot、过滤 enabled route 对应 model、按 quarantine state 排除 active quarantined model（read error fail-closed）、映射 compiled model 为 safe catalog entry、按 ID 排序输出；经 `internal/composition` 接入 runtime，`GET /v1/models` 返回 snapshot-backed catalog（不再 501）。

- Executor Phase 14 Responses：`services/executor/internal/responsecontract/AGENTS.md`。`internal/responsecontract` 为 OpenAI Responses API 的 provider-neutral 语义校验边界：`ValidateRequest` 以严格字段 allowlist 强制 stateless-only 执行（拒绝 `previous_response_id`/`conversation`/`store`/`background`/`include`/`moderation`/`prompt`/`truncation`/`service_tier`），校验 model/input/instructions/reasoning/tools/temperature/top_p/metadata/text/max_output_tokens；`ValidateResponse` 校验 bounded 结构（16 MiB wire cap、output ≤ 1024 items、content parts ≤ 256、usage ≤ 1e6 且一致、extension values ≤ 64 KiB）。CI race gate 新增 `./internal/responsecontract/...`。`POST /v1/responses` non-stream+stream 已 runtime 启用。

- Executor Phase 19 Auth JWT 集成：`services/executor/internal/jwtverifier/AGENTS.md`。`internal/jwtverifier` 提供 Ed25519 (EdDSA) 本地 JWT 验证，实现 `identity.Port`。`Verifier` 从 PKIX PEM 公钥文件加载 Ed25519 公钥，校验 EdDSA 签名、iss/aud/exp/nbf/sub/jti/role/token_version>=1；`Source` 实现 `LookupByKey`，role 映射 `"user"→RoleService`、`"admin"→RoleAdmin`，KeyID 留空（JWT 无 kid），Status=StatusActive（纯本地验证，不查禁用状态）。已知 trade-off：15min TTL 窗口内 revoked token 仍可用。`composition.Build` 中若 `cfg.JWTPublicKeyFile` 非空 → JWT source 优先，否则 → identityenv fallback；JWT 配置时 `EXECUTOR_IDENTITY_MAP_JSON` 变为 optional。启动 fail-fast：公钥文件缺失/格式错误 → `ErrJWTVerifier` → 拒绝启动。`internal/config` 新增 `JWTPublicKeyFile`/`JWTIssuer`/`JWTAudience` 字段。`nonstreamfacade`/`streamfacade`/`modelcatalogfacade` 的 `validPrincipal` 已更新为接受空 KeyID（JWT 无 kid）。OpenAPI 契约 `BearerAuth` description 更新为 JWT or API key。依赖 `github.com/golang-jwt/jwt/v5 v5.3.1`（与 Auth 相同版本）。CI race gate 新增 `./internal/jwtverifier/...`。
- Executor Phase 20 配置热重载：`services/executor/internal/configreload/AGENTS.md`。`internal/configreload` 提供 `Reloader`（LoadFile → Compile → validate → Publish，验证在发布前，失败保留旧 generation）；`configsource.CompileAndPublishNext` 支持 next generation 发布；`composition.Build` 返回 `*App`（`Handler` + `Reloader`）；SIGHUP 触发 on-demand reload；`EXECUTOR_CONFIG_RELOAD_INTERVAL`（默认 0 禁用）控制 stat mtime+size 轮询。零新依赖。CI race gate 新增 `./internal/configreload/...`。
- Executor Phase 21 /metrics 可观测性：`services/executor/internal/metrics/AGENTS.md`。`internal/metrics` 提供 Prometheus HTTP observability（counter vec + histogram vec + state gauges，低基数 label，匿名，与 healthz 同策略，不进 OpenAPI 契约，默认启用）；`quota.DomainInMemory.CountByState()` 与 `requestlog.InMemoryExecution.EventCount()` 提供 state gauge 数据；`internal/config` 新增 `MetricsEnabled`/`MetricsPath`（env `EXECUTOR_METRICS_ENABLED`/`EXECUTOR_METRICS_PATH`）；`composition.Build` 经 `metricsMux` 接入 /metrics，middleware 包在 AuthMiddleware 外层（401 也计数）。依赖 `github.com/prometheus/client_golang v1.24.0`。CI race gate 新增 `./internal/metrics/...`。
